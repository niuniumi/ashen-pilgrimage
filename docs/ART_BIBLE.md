# 《灰烬圣途 Ashen Pilgrimage》Art Bible

版本：v0.4.0-art-final-polish  
目标：Phaser 3 + Canvas 下的中世纪幻想像素风网页卡牌游戏成品化。

## 美术方向

关键词：中世纪幻想、温暖暗色、像素风、羊皮纸、黑铁、暗金、烛火、灰烬、古堡、墓园、修道院、炼金术、旧地图、暗红血迹、冷蓝护盾、绿色毒药、紫色诅咒。

禁止项：

1. 纯色大矩形直接当背景或 UI。
2. 默认按钮感、后台系统感、网页表单感。
3. 简单圆形/矩形拼成人物。
4. 大块空白和无纹理平面。
5. 无层级文字堆叠。
6. UI 风格各页不一致。

## UI 语言

1. 标题：金色大字，黑色投影，轻微外发光。
2. 面板：暗褐半透明底、双层暗金描边、角饰、轻微颗粒。
3. 按钮：黑铁底、暗金边、hover 烛火光、click 下沉。
4. 卡牌：羊皮纸底、职业色边框、插画窗、手稿纹理。
5. 地图：旧羊皮纸、烧焦边、手绘墨线、徽章节点。
6. 弹窗：羊皮纸内页、黑铁外框。
7. Tooltip：暗底金边，紧凑，不遮挡核心对象。
8. 图标：统一圆形徽章，暗金外圈。
9. 粒子：灰烬、火星、烛光、灵火，数量受控。

## 字体层级

1. 页面大标题：52-72 px。
2. 场景标题：34-46 px。
3. 面板标题：24-30 px。
4. 正文：18-22 px。
5. 注释：14-16 px。
6. 卡牌正文：12-14 px，必须清楚可读。

## 程序化资产标准

1. 所有角色、敌人、Boss 使用 `src/art/*Factory.js` 生成 Phaser Container/Graphics。
2. 主角至少 12 个视觉部件：轮廓、暗部、高光、职业色、武器/道具、阴影。
3. 敌人至少 8 个视觉部件，必须能从轮廓识别。
4. 不同角色不得共用同一身体模板换色。
5. 背景由天空、云、远山、古堡/教堂、地面、前景细节、暗角组成。
6. UI 控件和卡牌统一使用 Theme palette。
