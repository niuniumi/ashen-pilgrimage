# 《灰烬圣途 Ashen Pilgrimage》跨越级全面优化设计方案

版本基线：`v0.5.0-final-art-rescue`

## 1. 产品目标

当前版本已经是可运行、可部署、可试玩的 Phaser Canvas 垂直切片，但它的成品感仍主要停留在“功能完整的独立游戏原型”。下一阶段目标不应再是继续小修小补，而是升级为一个可以作为独立游戏 Steam Demo / itch.io Demo / 作品集主项目展示的完整体验。

最终目标：

1. 玩家 30 秒内理解世界、角色差异和核心玩法。
2. 玩家 10 分钟内形成一次完整 Build 方向。
3. 玩家 45-70 分钟完成一次第一章流程。
4. 每个职业有明确战术身份、视觉身份和剧情身份。
5. 每张卡牌、每个敌人、每个事件都服务于“灰烬圣变”的主题。
6. 视觉从代码生成 SVG 提升为正式立绘、卡牌插画、序列帧、分层场景和完整 VFX。
7. 音频从提示音提升为主题音乐、环境声、战斗声、UI 声、角色声效和动态混音。

## 2. 核心体验定位

一句话定位：

> 穿过被灰白圣火吞噬的中世纪王国，用卡牌构筑、仪式代价和角色信念，对抗逐渐腐坏的圣火教会。

体验关键词：

- 灰烬、圣火、赎罪、瘟疫、墓园、旧王权、教会腐化。
- 战斗不是纯数字交换，而是“选择代价”的仪式感。
- 三名角色不是皮肤差异，而是三套不同的信仰、机制和剧情视角。

竞品参考方向：

- 《Slay the Spire》：清晰构筑、敌人意图、可复玩路线。
- 《Darkest Dungeon》：压迫感、角色痛苦、哥特中世纪气质。
- 《Inscryption》：仪式感、卡牌触感、神秘叙事。
- 《Blasphemous》：宗教美术、痛苦圣像、压抑场景。

## 3. 当前版本主要短板

### 玩法层

1. 一章内容可玩，但长期构筑深度不够。
2. 三职业有机制，但职业专属路线还不够多。
3. 敌人行为和场景主题绑定还偏弱。
4. 事件数量可用，但缺少连续事件链和长期后果。
5. 遗物已经接入，但缺少套装、诅咒、职业专属遗物和稀有高光遗物。
6. Boss 有阶段，但战斗压迫、演出、专属机制仍不足。

### 内容层

1. 目前更像第一章 Demo，不像完整内容框架。
2. 世界观有方向，但缺少派系、关键 NPC、章节目标和多结局。
3. 角色剧情尚未深度绑定卡牌和事件。
4. 地图路线是功能型路线，缺少场景迁移和叙事推进。

### 视听层

1. v0.5 SVG 已明显好于几何图形，但仍不是正式美术。
2. 缺少正式主角立绘、动态战斗帧、敌人帧动画、Boss 演出。
3. 卡牌插画还是类型图形，不是叙事插画。
4. 场景没有分层视差、天气、局部光、体积雾、粒子层和特殊镜头。
5. 音效有基础反馈，但没有动态音乐、环境声和主题 leitmotif。

### UI/UX 层

1. UI 可用，但还没有“高端成品系统界面”的组件规范。
2. 地图、图鉴、商店、事件、奖励页视觉层级仍可继续升级。
3. 新手引导偏功能说明，缺少节奏化教学和视觉引导。
4. 可访问性、键鼠兼容、焦点状态和文本缩放需要系统设计。

## 4. 推荐产品结构

### 完整版本章节

建议做 3 幕 + 终局：

1. 第一幕：暮鸦村与墓园
   - 主题：瘟疫、墓葬、失控圣火余烬。
   - Boss：无首守墓骑士。
   - 目标：建立基础世界观和三职业核心玩法。

2. 第二幕：灰烛修道院
   - 主题：教会实验、忏悔仪式、烛印污染。
   - Boss：白蜡圣母。
   - 玩法重点：状态、延迟伤害、精神压迫、净化与腐化分支。

3. 第三幕：旧王都外墙
   - 主题：王权、空壳骑士、黑铁军团、灰火军械。
   - Boss：空心王胄。
   - 玩法重点：高护甲、高爆发、反击、场地危险。

4. 终局：灰白圣火源头
   - 主题：国王心脏、圣火真相、角色最终选择。
   - Boss：灰心之王 / 圣火心脏。
   - 玩法重点：构筑最终检验、多阶段仪式战。

### 单局流程

1. 角色选择。
2. 序章短叙事。
3. 幕地图选择路线。
4. 战斗 / 精英 / 事件 / 商店 / 休息 / 宝箱。
5. 幕 Boss。
6. 幕间选择：净化、献祭、强化、记忆。
7. 终局结算和剧情分支。

## 5. 玩法系统升级

### 三职业重构

#### 流亡骑士

定位：中速攻击、伤痕爆发、反击、处决。

升级方向：

- 伤痕从单一层数变为两种分支：
  - 裂伤：被撕裂时造成直接伤害。
  - 旧誓：被触发时获得护甲或反击。
- 新增“姿态”：
  - 盾卫：防御牌额外护甲。
  - 断誓：攻击牌额外伤害但失去护甲。
  - 追猎：击杀后抽牌或返还能量。
- 专属遗物：
  - 断誓王徽：每次撕裂 5 层以上伤痕，获得 1 能量。
  - 黑甲残片：首次受到致命伤时保留 1 点生命并获得 12 护甲。

立绘方向：

- 破损黑甲、褪色红披风、裂盾、短剑或阔剑。
- 头盔不开面，眼缝有灰白火光。
- 人物轮廓应稳、沉、压迫，动作像疲惫但仍有纪律的骑士。

#### 圣烛修女

定位：防御、烛印、延迟爆发、弱化敌人。

升级方向：

- 烛印分支：
  - 明烛：回合末燃烧造成伤害。
  - 罪烛：敌人攻击时反噬。
  - 白烛：燃烧时为玩家回复或护甲。
- 新增“祷文”系统：
  - 连续使用技能牌推进祷文段落。
  - 完成三段祷文触发强力效果。
- 专属遗物：
  - 熄灭圣铃：每场战斗第一次烛印爆发额外触发一次。
  - 忏悔面纱：每次给予虚弱，随机给另一名敌人 1 层烛印。

立绘方向：

- 黑白修女服、白蜡头冠、长烛杖、金色圣纹。
- 面部可保持神秘，但必须有情绪：冷静、慈悲、疲惫。
- 动画重点：烛火摇曳、袍摆、祷告手势。

#### 灰血炼金师

定位：自伤、药剂、腐蚀、低血量爆发。

升级方向：

- 灰血阈值：
  - 高血：稳定药剂、护甲、抽牌。
  - 半血：爆发药剂、腐蚀强化。
  - 低血：危险高收益，但每回合有失控风险。
- 新增“药剂槽”：
  - 每场战斗最多携带 3 个药剂标签。
  - 卡牌可调配、消耗、污染药剂槽。
- 专属遗物：
  - 裂纹量杯：每三次自伤，随机生成一张 0 费药剂。
  - 黑胆试剂：低于 35% 生命时，腐蚀伤害翻倍但治疗减半。

立绘方向：

- 鸟嘴面具、皮革长外套、背包瓶架、绿色/黄色药液。
- 身形应偏瘦、机敏、危险，不要像圆胖商人。
- 动画重点：药瓶晃动、毒雾、手部快速投掷。

### 新增卡牌规模

目标完整 Demo 建议：

- 每职业 35-45 张。
- 通用 35 张。
- 状态/诅咒 12-18 张。
- 总可见卡池约 150-185 张。

卡牌设计原则：

1. 每职业至少 4 个构筑方向。
2. 每个构筑方向有低费基础牌、中费核心牌、高费终结牌、遗物协同牌。
3. 所有卡牌描述统一格式，避免机制歧义。
4. 每张牌必须有明确插画主题，不允许只画类型符号。

### 遗物系统升级

建议规模：

- 普通遗物 45。
- 稀有遗物 30。
- 职业专属遗物每职业 12。
- 诅咒遗物 15。
- Boss 遗物 12。
- 总数约 138。

新增遗物类型：

- 代价遗物：强力，但增加诅咒或事件风险。
- 派系遗物：教会、王权、墓园、炼金、异端。
- 套装遗物：集齐 2-3 个后触发额外效果。
- 剧情遗物：改变事件选项和结局。

### 敌人与 Boss

普通敌人扩展到每幕 10-12 个，精英每幕 4 个，Boss 每幕 1 个，终局 Boss 1-2 个。

敌人设计必须绑定场景：

- 暮鸦村：村民、黑犬、墓园骷髅、乌鸦、瘟疫鼠群。
- 修道院：烛修士、忏悔者、圣蜡傀儡、失声唱诗班。
- 王都外墙：空壳骑士、王室弩手、灰火炮兵、断头执旗手。
- 终局：圣火残影、王心碎片、灰白天使。

Boss 标准：

1. 每个 Boss 至少 3 阶段。
2. 每阶段有视觉变化、音乐变化和行动池变化。
3. 进入新阶段必须有镜头、屏幕特效和 UI 横幅。
4. Boss 战应有专属机制，不只是更高数值。

## 6. 剧情与世界观扩展

### 派系

1. 圣火教会：维持秩序但制造灾难。
2. 旧王庭：追求永生，牺牲王国。
3. 灰烬异端：相信灰烬是新生。
4. 暮鸦幸存者：普通人视角，提供道德选择。
5. 炼金学会：研究圣火污染，和灰血炼金师有关。

### 角色个人线

流亡骑士：

- 曾是王庭骑士，参与封锁暮鸦村。
- 核心冲突：服从旧誓还是承认罪责。
- 结局分支：殉誓、赎罪、篡火。

圣烛修女：

- 曾参与圣火净化仪式，知道教会真相。
- 核心冲突：信仰是否仍值得守护。
- 结局分支：净火、灭火、成为新圣母。

灰血炼金师：

- 用自己身体试验灰血解药。
- 核心冲突：救人还是证明理论。
- 结局分支：自我献祭、制造解药、释放污染。

### 事件链

单点事件升级为事件链：

- 乌鸦送信 1/3：收到求救。
- 乌鸦送信 2/3：选择救人、交易、忽视。
- 乌鸦送信 3/3：根据选择获得遗物、诅咒、商店折扣或 Boss 弱化。

推荐事件链：

1. 忏悔室。
2. 瘟疫病床。
3. 旧王密令。
4. 炼金学会残页。
5. 黑犬主人。
6. 白蜡圣母的孩子。

### 结局

至少 6 个结局：

1. 熄灭圣火。
2. 净化圣火。
3. 继承王心。
4. 灰烬新生。
5. 角色专属坏结局。
6. 真结局：三职业线索汇合后解锁。

## 7. 视觉升级方案

### 美术资产目标

当前 SVG texture 只能作为临时正式版。下一阶段必须建立真实资产管线：

1. 主角全身立绘：每角色 1 张选择页大立绘、3 张剧情表情差分。
2. 战斗 sprite：每角色 idle、attack、defense、hit、skill、death。
3. 敌人 sprite：每敌人 idle、attack、hit、death。
4. Boss 大图：每阶段 1 张主体 + 2-3 个局部特效层。
5. 卡牌插画：每张牌独立 512x384 原画，游戏内裁切到卡面。
6. 场景背景：每幕 4-6 张分层背景。
7. UI：完整九宫格面板、按钮、卡框、地图节点、遗物图标、状态图标。

### 画风标准

推荐方向：暗黑中世纪手绘像素混合风。

- 角色：手绘轮廓 + 像素化边缘处理。
- 场景：高分辨率分层绘制，游戏内加轻微像素滤镜。
- UI：黑铁、暗金、旧羊皮纸、蜡封、圣纹。
- 色彩：深紫黑、焦褐、灰白火、暗金、血红、毒绿。避免全屏单一暗色。

### 场景重构

主菜单：

- 不再只是远山和火堆。
- 加入可动的篝火、风中灰烬、远处教堂钟影、标题背后的灰白圣火。
- 菜单按钮像铁质圣匣嵌在右侧。

角色选择：

- 三角色以半身立绘或全身立绘为第一视觉。
- 卡片不只是信息框，应像教会档案/通缉书/遗物卷宗。
- 鼠标悬停时展示角色专属背景短动画。

战斗：

- 每幕使用不同战斗背景。
- 玩家和敌人站在明确地面透视线上。
- 攻击时镜头轻推、斩击粒子、血光、护甲碎裂。
- Boss 战切换为专属舞台，不复用普通战斗背景。

地图：

- 路线图升级为可滚动羊皮卷或立体旧地图。
- 节点有不同蜡封、墨迹、火光状态。
- 完成路线留下灰烬足迹或烧灼痕。

图鉴：

- 从普通列表升级为“灰烬手册”。
- 卡牌、敌人、遗物有大图、故事短句、机制说明。

## 8. UI/UX 全面升级

### 设计系统

建立 `Ashen UI Kit`：

- 字体层级：标题、章节标题、面板标题、正文、说明、数值、警告。
- 色彩 token：背景、面板、边框、主文字、次文字、危险、护甲、能量、稀有度。
- 间距 token：8/12/16/24/32。
- 组件：按钮、面板、弹窗、卡牌、血条、状态徽章、日志、tooltip、toast、地图节点、遗物格。

### 主要界面改版

主菜单：

- 增加“继续旅途”的存档摘要。
- 新旅程按钮必须是第一主 CTA。
- 设置、图鉴、制作组作为次级入口。

角色选择：

- 加入职业难度、机制示意、小型起始牌组预览。
- 选择后右侧展示“玩法路线建议”，不是让玩家读长段文字。

战斗：

- 手牌区强化层级：抽牌堆/弃牌堆/消耗堆、能量、状态、提示要一眼可见。
- 敌人意图必须颜色编码：攻击红、防御蓝、强化金、削弱紫、特殊灰白。
- 日志缩短，重要反馈前置到战场动画和飘字。

奖励页：

- 卡牌奖励要有比较功能。
- 新卡与当前构筑的关联要被提示，例如“与伤痕协同”。

商店：

- 分区：卡牌、遗物、移除、治疗、特殊交易。
- 明确当前金币和购买后剩余金币。

事件：

- 每个选项显示代价、收益、风险。
- 高风险选项用更强视觉警告。

设置：

- 音量分主音量、音乐、音效、UI。
- 动画速度、屏幕震动、闪烁强度可调。
- 色弱模式和大字体模式。

## 9. 视听盛宴方案

### 音乐

建立动态音乐层：

1. 主菜单：低沉合唱、远钟、火焰噼啪。
2. 地图：低频 drone、羊皮纸翻动、风声。
3. 普通战斗：鼓点 + 弦乐 ostinato。
4. 精英战：增加低铜管和教会钟声。
5. Boss 战：三阶段音乐叠层，第三阶段加入合唱。
6. 休息点：温暖但不安全的烛火旋律。

### 音效

音效分层：

- UI：按钮金属声、羊皮纸翻页、蜡封破裂。
- 卡牌：抽牌、悬停、选中、飞行、打出、消耗。
- 战斗：剑击、盾挡、火焰、毒液、骨裂、布料、怪物叫声。
- 环境：风、灰烬、远钟、乌鸦、低语。

### VFX

核心特效：

- 斩击：方向性刀光 + 屏幕轻震 + 火星。
- 护甲：蓝灰盾弧 + 金属格挡声。
- 烛印：敌人身上小蜡火，触发时白火爆开。
- 灰血：角色血雾、毒绿药雾、瓶子碎裂。
- Boss 阶段：暗幕、镜头拉近、灰白火焰喷发。

### 镜头

Phaser Canvas 内也可以做镜头语言：

- 攻击命中：0.08-0.16 秒轻推或震动。
- Boss 登场：黑幕、缩放、慢速灰烬。
- 精英死亡：短暂慢动作。
- 低血量：边缘红暗角、心跳低频。

## 10. 技术实现路线

### 架构升级

1. 数据层继续保持 `src/data/*`，但拆分章节内容：
   - `data/acts/act1`
   - `data/acts/act2`
   - `data/acts/act3`
2. 建立内容 schema 校验：
   - 卡牌字段完整性。
   - 敌人行动合法性。
   - 事件选项代价/奖励合法性。
3. 建立资产 manifest：
   - 每个角色、敌人、卡牌插画、音频都有 id、路径、尺寸、用途。
4. 战斗动画事件总线：
   - BattleSystem 只产出事件。
   - BattleScene 消费事件并播放动画。
5. 音频管理器扩展为 mixer：
   - music、ambience、sfx、ui 四条总线。

### 性能策略

- 大图使用 WebP/AVIF，保留 PNG 源文件。
- 战斗 sprite 使用 spritesheet atlas。
- UI 使用九宫格或 texture atlas。
- 预加载按场景分批，不要一次加载全部章节。
- 低端模式关闭部分粒子、阴影、后处理。

### QA 自动化

每次大版本必须截图：

- 主菜单。
- 角色选择三角色。
- 地图。
- 普通战斗三角色。
- 精英战。
- Boss 三阶段。
- 奖励、商店、事件、休息、图鉴、设置。
- 1280x720、1366x768、1536x864。

每次内容扩展必须测试：

- 三职业完整通关。
- 每个 Boss 阶段切换。
- 每张卡牌效果。
- 每个事件分支。
- 每个遗物触发。
- 存档兼容。

## 11. 阶段计划

目标：让核心战斗和 UI 先达到成品观感。

目标：角色形象成为第一卖点。

内容：

- 三角色轮廓、色彩、姿态明显不同。
- 卡牌机制与角色视觉一致。

### Phase 3：第一幕完整化

目标：把第一章从 Demo 变成成熟章节。

内容：

- 第一幕敌人扩到 12 普通、4 精英、1 Boss。
- 第一幕事件链 6 条。

- 至少 6 种有效构筑。
- Boss 战具有记忆点。

### Phase 4：第二幕与第三幕

目标：形成完整 Roguelike 内容规模。

内容：

- 三职业都有不同路线偏好。
- 玩家死亡和胜利都有继续游玩的动力。

### Phase 5：终局、结局与发布打磨

目标：形成可公开发布 Demo 或小体量商业版。

内容：

- 终局 Boss。
- 多结局。
- 图鉴完善。
- 新手引导重做。
- 平衡、性能、无障碍、存档、部署 QA。

验收：

- 首次玩家无需解释即可开始。
- 重要流程无白屏、无错位、无文本裁切。
- 完整通关稳定。
- 线上版本可被公开分享。

## 13. 不建议继续做的事

1. 不建议继续靠 Phaser Graphics/SVG 拼角色作为最终美术。
2. 不建议继续只扩卡牌数量而不增加构筑路线。
3. 不建议继续把日志作为主要反馈渠道。
4. 不建议在没有设计系统的情况下逐页美化。
5. 不建议只做暗色和金边，否则会变成单调主题。
6. 不建议把剧情只放在开场和结尾，事件、遗物、卡牌都应承载叙事。

## 14. 最终验收标准

视觉：

- 任意主界面截图可以公开展示。
- 三主角立绘具有辨识度和角色魅力。
- 卡牌插画不再是类型图标。
- Boss 战截图有压迫感。

玩法：

- 每职业至少 4 条构筑路线。
- 每幕至少 1 个独特机制。
- 每个 Boss 有专属机制和视觉阶段。
- 单局有明显成长曲线。

内容：

- 至少 3 幕 + 终局。
- 至少 150 张可用卡牌。
- 至少 120 个遗物。
- 至少 45 个事件。
- 至少 40 个敌人。

视听：

- 每个主要场景有独立音乐或氛围层。
- 每类行动有专属音效。
- 战斗反馈不依赖文字日志。
- 玩家能通过声音判断关键状态变化。

技术：

- `npm run build` 稳定通过。
- 自动截图 QA 覆盖主流程。
- 线上 smoke 通过。
- 存档兼容策略明确。

## 15. 结论

《灰烬圣途》下一阶段的重点不是“再加一点内容”，而是把已有垂直切片升级为正式产品管线：设计系统、资产系统、动画系统、音频系统、内容 schema、章节结构和自动 QA 必须一起推进。

推荐执行顺序是：

1. 先重做战斗体验和 UI 系统。
2. 再确定三主角正式美术。
3. 然后完整化第一幕。
4. 最后扩第二幕、第三幕和终局。

这样不会陷入“内容越来越多但仍像原型”的问题，也能保证每一轮投入都真正提升成品感。
