# v1.1.0 Production

## v1.1.1 稳定性更新

- 将局内存档升级到 v3，可识别旧 7 层地图、旧坐标范围和损坏的节点引用。
- 旧路线会以同一种子重建为 12 层，同时保留角色、生命、金币、牌组、遗物和已完成层数。
- 保留可恢复的战斗断点；孤立的活动节点会安全回滚，避免重复结算。
- 地图节点和路线改为首帧可见，并增加坐标钳制和明确渲染深度，防止动效或旧数据导致空白地图。
- 新增真实浏览器迁移回归测试，并纳入 GitHub Actions 持续集成。
- 清理暂停菜单中的重复废弃实现，减少后续维护歧义。
- Netlify 额度耗尽后将正式试玩迁移到 GitHub Pages，并增加主分支自动发布流程。

发布日期：2026-07-11

## 主要变化

- 修复失败结算墓碑素材的绿幕残留，增加阴影、雾层和入场反馈。
- 每章由 9 层扩充至 12 层，重新安排战斗、精英、事件、商店和休息节奏。
- 替换旧 BGM 路由，提供菜单、三章地图、三章战斗和首领共 8 条独立曲目。
- 效果音扩充为 50 个样本，并按交互类型轮换播放。
- 增加打击停顿、冲击爆点、卡牌飞行动势和首领转阶段演出。
- 修复首领蓄力攻击立即结算的问题，改为预警一回合后释放。
- 将洗牌、奖励、商店、宝箱和随机遗物统一接入局内种子，确保续玩结果可复现。
- 更新公开项目文档、音频授权清单和持续集成流程。

## 兼容性

- 保持 v2 存档格式和旧存档迁移逻辑。
- 动画仍可在设置中关闭。
- BGM 同时提供 OGG 与 MP3，降低浏览器解码差异。
