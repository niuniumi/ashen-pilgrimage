# v2.0.0 Pixel Rebuild

## 核心修复

- 修复战斗胜利后奖励结算可能重复执行、返回地图后节点锁死的问题。
- 奖励页刷新后可准确恢复，不再回滚活动节点或跳过未领取奖励。
- 宝箱、休息、事件和奖励采用单次事务结算，刷新或连点不再重复获得收益。
- v4 存档可以恢复誓约、战斗、首领入场、奖励、事件、商店、休息、宝箱、章节结算和结果页。
- 修复快速切换场景时延迟资源重复加载与纹理键冲突。
- 修复战斗中主动返回地图残留活动节点，同时保留已胜利待领取的奖励事务。
- 新增真实点击回归：新旅程、选角、立誓、首战、重复结算、奖励页刷新、领奖、回地图、进入下一节点。

## 像素重构

- 所有生产页面统一为像素风，移除旧手绘、SVG、占位图和混搭图集。
- 新增六张生产背景、本地中文像素字体、4 px 网格与有限色板。
- 新增 16 个透明角色资产与 4 x 4 语义敌人图集，完整映射三名主角、28 种敌人与三名 Boss。
- 重构面板、按钮、卡牌、图标、状态条、弹窗、序章、战斗特效与失败墓碑。
- 地图继续保留三章共 36 层，不缩减关卡内容。

## 视听与性能

- 替换 8 条 CC0 场景 BGM，剧情、休息、失败不再复用菜单曲。
- 替换 51 个高频 UI、卡牌、武器、格挡、金币和书页效果音样本。
- 统一响度、限幅、音效增益和场景淡入淡出。
- 生产包由约 169 MB 降至约 35 MB。

## 质量门禁

- 76 项单元测试。
- 内容、24 项像素资产和视觉绑定审计。
- 180 场确定性战斗模拟，覆盖三名角色、普通战、精英战与首领战。
- 旧存档地图迁移浏览器回归。
- 5 类断点恢复、4 类事务保护、结算与下一节点真实点击浏览器回归。
- 31 步发布流程和 32 张流程截图回归。
- 修复 GitHub Project Pages 字体路径，并为字体等待、资源并发和失败重试设置启动保护。
