# v2.3.0 发布说明

发布日期：2026-07-16

## 体验与内容

- 完成像素化胜负结算场景，强化胜利、失败、角色结局与本局统计的视觉层级。
- 收紧宝箱、事件、休整与商店等非战斗场景的选择状态，避免重复结算和输入穿透。
- 保持 3 名角色、70 张卡牌、24 件正式遗物、28 种敌人、21 个事件、9 条角色结局和三章 36 层路线的正式内容规模。

## 资源与稳定性

- 按场景和章节加载战斗、地图与角色资源，生产像素图统一为无损 WebP。
- 加固运行时图片生成、关键资源恢复和加载失败重试，恢复期间暂停场景输入，避免状态被提前推进。
- 修正本地中文字体的 Vite 公共资源路径，根路径和 `/ashen-pilgrimage/` Pages base 均由构建器正确重写。
- Phaser 与启动必需模块继续使用单体启动 bundle，不进行高风险动态导入改造；Vite warning 边界限定为 1600 KB，并保留 gzip 体积报告。

## 发布工程

- CI 统一使用 Node.js 24 与 pnpm 11.7，覆盖素材校验、单元测试、静态契约、确定性模拟、生产构建和完整浏览器发布流。
- GitHub Pages 仅在 `main` 的 CI 成功后部署对应 `head_sha`；手动部署入口保留，部署完成后执行线上 `qa:deploy-smoke`。
- 新增 release-config 合同测试和[生产验证手册](PRODUCTION_VERIFICATION.md)，固定版本、构建、QA 参数与部署职责。

## AI 辅助开发说明

本项目保持“AI 辅助开发”的真实表述：项目作者负责产品方向、玩法与审美取舍、验收和发布决策；OpenAI Codex 辅助实现、测试自动化、素材治理与部署配置。发布结论以可复现测试和构建证据为准，不以生成内容本身作为完成证明。
