# Visual Acceptance

## 验收原则

- 游戏主界面必须像 Canvas 游戏，不像普通 DOM 网页卡片。
- 背景不得包含不可交互假按钮、假卡牌、假血条或旧菜单文字。
- 参考图不得整张或随意裁切进游戏 UI。
- 16:9 缩放保持不拉伸、不压扁、不变形。
- 所有可交互 UI 必须是 Phaser 游戏对象。
- UI 不得显示内部英文 id。

## 当前视觉实现

- 主菜单：程序化暮色天空、远山、古堡剪影、篝火和灰烬粒子。
- 指南与地图：羊皮纸色调，固定游戏坐标布局。
- 角色选择：三张大型 Phaser 角色卡，程序化像素剪影。
- 战斗：玩家左侧、敌人右侧、手牌底部、战斗日志右侧、意图在敌人头顶。
- 卡牌：Phaser Container，包含费用、中文名、插画区、描述、类型和稀有度。
- 动画：卡牌飞行、斩击、受击、护盾、治疗、伤害数字、回合横幅。

## 自检标准

每张 QA 截图需检查：

1. 是否像游戏。
2. UI 与背景是否融合。
3. 是否存在假 UI 残留。
4. 是否存在错位或拉伸。
5. 是否出现内部英文 id。
6. 玩家是否知道下一步操作。
7. 动画和反馈是否可见。
