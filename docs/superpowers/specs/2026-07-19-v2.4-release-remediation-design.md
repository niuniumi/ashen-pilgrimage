## 设计方向：黑铁圣书 / 深暗像素手稿

- 保留像素羊皮纸地图、煤黑面板、方角黑铁框、PixelCN 字体与现有主场景插画。
- 色彩继续以 `#08090d`、`#11131a`、`#1b1d24` 为底；骨白为正文；暗金表示结构；亮烛金只表示焦点/可选路径；血红只表示危险；甲胄蓝表示主要确认或防御。

## 可靠性契约

1. 地图只有当前可选节点接收 pointer/手型；至少 100×100 逻辑像素命中区；中心与四个边缘均必须进入同一节点。
2. 战斗进入 `reward`、`act-clear` 或 `result` 结算事务后，不得通过暂停菜单跳回地图破坏状态；返回主菜单必须原样保存并可继续恢复。
3. 旧存档的 `pendingScene` 与 `activeNode` 必须同时有效且节点类型相容；无法恢复时安全回滚地图，不能免费领取非战斗奖励。
4. 缺失数组字段必须由迁移器补齐；未知卡牌/遗物 ID 或错误字段类型必须被拒绝或安全修复，不能启动后空白。
5. 所有用户可见长中文正文必须按真实字宽换行并遵守常见中文行首/行尾禁则；序章四页不得越过正文区或压住页码。

## 交互、无障碍与响应式

- 地图支持方向键/WASD 在可选节点间移动、Enter/Space 确认；序章支持左右键翻页、Enter 继续、Escape 跳过；角色选择保留已有键盘控制。
- 场景标题、当前选项、toast 与战斗关键状态同步到常驻 `aria-live`；核心场景提供屏幕外真实按钮供读屏激活。
- 手机竖屏小视口显示同风格“请旋转设备”遮罩并阻止误触；横屏继续采用 16:9 FIT，主要触控区目标约 44 CSS px。
- pointer down、hover、focus、confirm 都有明确但克制的视觉/声音反馈；锁定态不得显示手型或焦点光。

## 动效与听觉

- 现有 `animation` 设置仍保持兼容；系统 `prefers-reduced-motion` 在旧设置缺字段时生效。
- 统一 MotionPolicy：full 使用 80–170ms 微交互、260–460ms 场景过渡；reduced 直接落最终状态，不保留循环呼吸、打字机、镜头淡入或 hover tween。
- 保留 v2.4 SFX 分组、冷却、轻微速率变化与瞬时 BGM ducking；首次 pointer 或 keyboard 手势都能解锁音频。
- 页面隐藏/pagehide 时暂停 BGM；恢复可见后仅在用户允许音乐且音频已解锁时恢复，并显式恢复 suspended AudioContext。
